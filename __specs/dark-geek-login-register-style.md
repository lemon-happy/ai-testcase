# 暗黑极客风登录注册页面样式

## 1. 需求摘要 (Summary)

当前登录和注册页面采用标准的浅色简约风格，缺乏视觉冲击力与个性化辨识度。本次改版旨在将页面整体视觉语言切换为「暗黑极客」风格，以动态海浪粒子作为全屏背景，提升产品的科技感与沉浸感，同时保持良好的可读性与操作流畅度。

## 2. 功能需求 (Functional Requirements)

### 2.1 全屏动态海浪粒子背景

- 页面背景为全屏动态粒子效果，粒子的运动轨迹模拟海浪起伏形态（正弦波/余弦波叠加）。
- 粒子颜色主色调为青色/电子蓝（如 `#00ffcc`、`#0ff`），在深色背景（接近纯黑 `#0a0a0f`）上形成高对比度的发光感。
- 粒子之间在距离阈值内通过细线相连，营造「网络节点」的极客氛围。
- 粒子数量与连线密度应适中，不影响页面的整体性能感知（无明显卡顿）。
- 动效持续循环，无任何交互触发。

### 2.2 登录表单样式

- 表单容器背景为半透明深色磨砂玻璃效果（`glassmorphism`），使粒子背景透过表单隐约可见。
- 表单边框为细线发光描边，颜色与粒子主色调呼应（青色/电子蓝细线 + 微弱外发光）。
- 所有输入框采用深色背景 + 半透明，获得焦点时边框高亮（青色发光）。
- 按钮采用青色/电子蓝渐变填充，悬浮时有发光扩散动效。
- 文字颜色主色调为亮白色或浅灰色，确保在深色背景下高可读性。
- 页面标题/品牌标识保留，字体风格改为等宽/像素风字体以强化极客感。

### 2.3 注册表单样式

- 与登录页保持视觉语言一致（相同的粒子背景、玻璃态容器、配色方案）。
- 表单结构不变（字段数量、验证逻辑不在本次范围内）。
- 多行表单项在深色风格下保持清晰的视觉层次，避免因字段密度导致的视觉混乱。

### 2.4 交互动效细节

- 页面加载时，表单容器以淡入+向上轻微位移的方式出现（不超过 0.5s）。
- 输入框在获取焦点时，边框发光强度提升，有平滑过渡动效。
- 提交按钮在鼠标悬浮时，发光晕染向外扩散，点击时有轻微按压缩放反馈。

## 3. 边界条件与异常分支 (Possible Edge Cases)

- **低性能设备**：部分设备 GPU 性能较弱，粒子动画可能导致帧率下降。需确保降级策略（减少粒子数量或暂停动效）不破坏基本可用性，表单始终正常显示。
- **移动端适配**：粒子背景在小屏幕设备上需自适应，表单宽度随视口缩放，不出现横向滚动条。
- **弱视/高对比度偏好**：表单核心内容（标签、输入框文字、按钮文字）必须满足 WCAG AA 级对比度要求（文本与背景对比度 ≥ 4.5:1）。
- **浏览器兼容性**：`backdrop-filter`（磨砂玻璃效果）在部分旧版浏览器中不被支持，需有无磨砂的纯深色背景兜底方案，保证表单可读。
- **动效偏好设置**：用户系统若设置了 `prefers-reduced-motion`，粒子动画应停止或降为静态粒子图，表单仍保持完整样式。

## 4. 验收标准 (Acceptance Criteria)

### 背景粒子效果
- [ ] 访问 `/login` 和 `/register` 页面时，全屏区域展示动态粒子背景，无白屏或背景缺失现象。
- [ ] 粒子运动轨迹呈现连续的海浪起伏形态，视觉上有明显的波动感。
- [ ] 粒子颜色为青色/电子蓝系，背景底色为深色（接近纯黑），整体呈现高科技发光风格。
- [ ] 粒子之间存在连线，连线会随粒子移动动态伸缩消失，呈现网络节点效果。
- [ ] 粒子动画持续不间断循环，页面停留 30 秒以上无卡顿或异常停止。

### 表单视觉风格
- [ ] 登录表单容器呈现半透明磨砂玻璃质感，粒子背景透过容器隐约可见。
- [ ] 表单容器边框为青色/电子蓝细线，带有微弱外发光效果。
- [ ] 所有输入框底色为深色半透明，文字颜色为亮白或浅灰，标签清晰可读。
- [ ] 输入框获取焦点时，边框颜色高亮且有发光扩散效果，过渡动画流畅（无闪烁）。
- [ ] 提交按钮底色为青色/电子蓝渐变，文字颜色与按钮背景对比度 ≥ 4.5:1。
- [ ] 注册页面与登录页面在视觉语言上保持一致，无明显风格断层。

### 交互动效
- [ ] 页面首次加载时，表单容器有淡入动效，动效时长不超过 500ms。
- [ ] 鼠标悬浮提交按钮时，发光晕染向外扩散，视觉效果明显。
- [ ] 点击提交按钮时，按钮有轻微缩放反馈，持续时间不超过 200ms。

### 响应式与兼容性
- [ ] 在 375px 宽度（iPhone SE）的移动端视口下，表单完整显示，无元素溢出或横向滚动。
- [ ] 在不支持 `backdrop-filter` 的浏览器中，表单容器以纯深色不透明背景显示，所有字段仍清晰可操作。
- [ ] 当系统设置 `prefers-reduced-motion: reduce` 时，粒子动画停止或静止，表单样式完整保留。

### 原有功能无回归
- [ ] 登录表单的邮箱/密码输入、提交、错误提示、跳转注册页等功能均正常工作。
- [ ] 注册表单的所有字段输入、提交、错误提示、跳转登录页等功能均正常工作。
- [ ] 已登录用户访问 `/login` 或 `/register` 时，仍会被正确重定向至 `/dashboard`。

## 5. 待确认问题 (Open Questions)

- 粒子数量的具体范围是否有性能基准要求（例如：移动端最多 60 个粒子，桌面端最多 150 个）？可以
- 品牌标识/Logo 是否需要在暗黑风格下做相应的颜色适配（如改为白色或发光版本）？需要
- 是否需要在登录/注册页之间切换时保持粒子背景不重新初始化（连续动效体验）？需要
- 「等宽/像素风字体」是否有具体的字体家族偏好（如 `JetBrains Mono`、`Fira Code` 等）？没有

## 6. 测试指南 (Testing Guidelines)

请在对应的测试目录（如 `./tests`）中为新功能创建测试文件，并重点针对以下核心场景编写高价值的测试用例（避免过度测试）：

- **视觉回归测试**：对登录页和注册页截图，与基准图比对，确保粒子背景容器、表单容器、输入框、按钮的关键样式属性（颜色、边框、透明度）未意外变更。
- **交互功能测试**：在新样式下完整走通登录流程（成功登录、错误密码）和注册流程（成功注册、字段校验失败），确保功能无回归。
- **响应式测试**：模拟 375px、768px、1280px 三种宽度，验证表单无溢出、布局无破坏。
- **无障碍测试**：使用自动化工具（如 axe-core）检测登录和注册页面，确保无 WCAG AA 对比度违规。

## 7. 非目标范围 (Out of Scope)

- 登录和注册表单的字段结构、校验逻辑、API 接口不在本次修改范围内。
- Dashboard 及其他页面的样式不受本次改动影响。
- 不引入新的用户认证机制或权限逻辑。
- 不涉及任何后端代码变更。
- 不包含深色/浅色主题切换开关功能（本次直接固定为暗黑风格）。
- 不涉及具体的 Canvas API、WebGL 技术选型或 CSS 属性的实现细节。
